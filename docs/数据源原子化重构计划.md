## 1. 背景与目标

### 1.1 一句话
把数据层从「**门面 datahub + 3 层抽象(adapter/manager/fetcher)+ 源散落 + 整合库黑盒**」重构成
「**门面 datahub(不变)+ 路由/降级(不变)+ 一家真源一个原子模块(直连、归一、无整合库)**」。

### 1.2 现在乱在哪(3 个病灶)
1. **同一家源散在 3-4 个文件**。例:**东财** 出现在 `a_stock_data_adapter`(push2/push2his/datacenter)+ `datahub` 内联(`_kline_eastmoney`/`_fund_nav_eastmoney`)+ `data_source_manager`(curl 东财)。想改"怎么调东财"要动 3 处。新浪、akshare 同样散落。
2. **路由套路由(嵌套降级)**。`datahub.kline(raw)` 的 `_route` 挂 `[fetcher, baostock, east, mootdx]`,而其中 `fetcher`(StockDataFetcher)**自己又是一条 8 源降级链**(→ `data_source_manager.get_stock_hist_data` → 新浪/东财/akshare/tushare/pywencai/yfinance/Ashare/mootdx)。结果 east/mootdx **在内外两层重复出现**、链路 4-5 跳。
3. **三层抽象边界模糊**:`adapter`(直连)← `manager`(包 adapter + 库源降级)← `fetcher`(包 manager + 技术指标)← `datahub`(全包 + 自己还内联)。谁包谁说不清。
4. **整合库黑盒**:`adata`/`akshare`/`tushare` 是第三方「整合体」,内部又偷偷包东财/新浪/同花顺等真源,**你不知道也控制不了它实际打哪**;它的源一断你受制于人。

### 1.3 目标收益
- 加新源 = 新建 1 个文件 + 在对应域的 `_route` 列表里登记 1 行。
- 改一家源 = 只动 1 个文件。
- 链路扁平(门面 → 路由 → 直连源,无嵌套降级)。
- 去掉整合库黑盒,直连真源,字段/单位/复权口径完全自控。

## 2. 设计原则(铁律)

1. **datahub 不变**:它仍是全项目唯一数据门面;`_route` 健康度路由 / 全源熔断 / 硬超时 / 三级缓存全保留。**降级只在 `_route` 这一层**。
2. **原子源模块**:一家真 provider 一个文件,**直连该家 API(HTTP/协议)、归一格式、内部不 import 任何整合库、自身不做跨源降级**。
3. **整合库处置(已拍板)**:
   - `adata` → **删**(二道贩子,真源都有直连)。
   - `akshare` → **降为各域末位兜底**(直连全挂时的安全网),绝不进主路径。
   - `tushare` → **保留为可选源**,但 **无 token 时 `available()=False`、根本不尝试**(不再像现在无 key 还走降级链空转)。
4. **格式契约**:所有源返回统一 schema(见 [§5.2](#52-归一契约)),下游无差别。重写直连必须**逐字段对齐**原有口径(volume 单位「股」、复权口径 raw/qfq、K线 `DatetimeIndex='Date'` + 大写 `Open/Close/High/Low/Volume`)。

### 3.2 `_route` 机制(保留不动)
`_route(domain, [(源名, 无参thunk), ...], empty, timeout)`:
- **健康度排序**:成功率为主(`ok/(ok+fail)`)− 延迟轻罚(最多 −0.2);连续失败 ≥2 且 120s 内 → 额外 −1.0;**未知源中立 0.5**(新源不抢主位)。
- **全源熔断**:某域所有源都在活跃冷却(score<−0.5)→ 直接返 empty,秒级降级,冷却 120s 自愈。
- **硬超时**:每源独立线程 + `future.result(timeout)`,默认 20s(kline 45s),超时 cancel 孤儿线程切下一源。
- **线程池** `_ROUTE_POOL`(64 线程):防批量假超时。
- **三级缓存**:L0 进程内存 → L1 Redis(JSON 友好) → L2 本地 pickle(含 DataFrame)。**缓存在域级**(miss 才走 `_route`);K线另有专用磁盘缓存(raw/qfq 两套 + 3 天 TTL + 失败回退历史)。

### 3.3 嵌套降级实例(✅ 已摊平,2026-06-28)
**改前**(east/mootdx 在内层链 + 外层 `_route` **各出现一次**、链路 4-5 跳):
```
datahub.kline(code, '1y', adjust='raw')
  └─ _route("kline", [fetcher, baostock, east, mootdx])
       └─ fetcher = StockDataFetcher.get_stock_data
            └─ data_source_manager.get_stock_hist_data
                 └─ 降级链:新浪 → curl东财 → akshare → tushare → pywencai → yfinance → Ashare → mootdx
```
**改后**(一层直连原子源,无嵌套;StockDataFetcher 反过来委托 datahub.kline):
```
datahub.kline(code, '1y', adjust='raw')
  └─ _route("kline", [sina_raw, baostock, east, mootdx, tencent, akshare(, tushare)])   # 全部直连 sources/*
```

## 5. 原子源模块设计

### 5.1 目录结构
```
data/sources/
  _common.py        # 归一工具(见 §5.3)
  eastmoney.py  sina.py  tencent.py  ths.py  baidu.py  jsl.py  cls.py  cninfo.py
  baostock.py  mootdx.py  pywencai.py  tushare.py
  akshare.py        # 末位兜底层
data/datahub.py     # 门面:域函数 _route([sources.x.cap, ...]) + 缓存
```

### 5.2 归一契约(每模块按能力暴露,返这些格式)
- `quotes(codes) -> {code6: {price, change_pct, pe_ttm, pb, mktcap, name, ...}}`
- `kline(code, period, interval, adjust) -> DataFrame`(`DatetimeIndex='Date'` + 大写 `Open/Close/High/Low/Volume`,volume 单位股)
- `indices() -> [ {name, code, price, change_pct} ]`
- `financials(code, report_type) -> [dict]` · `valuation(code) -> dict` · `full_valuation(code) -> dict`
- `north_flow(days) -> [dict]` · `capital_flow(code, days) -> [dict]` · `dragon_tiger(date) -> [dict]` · `margin(code) -> [dict]`
- `sector_ranking/sector_fund_flow/sector_spot -> ...` · `news/announcements -> [dict]` · `convertible_bonds() -> [dict]` · `fund_nav(code) -> DataFrame`
> 模块只实现该 provider **真能提供**的能力(东财能力最多;腾讯只 quotes/indices)。

### 5.3 `_common.py` 归一工具(从现有散落处收口)
- `norm_code(code) -> '600519'`(剥前缀/补零取后 6 位)
- `em_secid(code) -> '1.600519'`(东财 secid,沪 1/深京 0,处理 900沪B/920北交/科创 688)
- `bs_code(code) -> 'sh.600519'`(baostock)、`sina_code`/`tencent_code` 各家前缀
- 列名/单位归一:`to_ohlcv(df)`(大写 OCHLV + DatetimeIndex 'Date' + volume→股)
- `throttle(source)`(复用 `rate_limiter`)、`ak_safe(fn, *a)`(复用 `akshare_safe`,**仅 akshare 兜底层用**)
- `http_get_json(url, headers, timeout)`(统一 UA/超时/重试,直连源共用)

### 5.4 模块契约(每个 `sources/*.py` 遵守)
- 只直连本家 provider;**禁止 `import akshare/adata/tushare`**(akshare.py/tushare.py 除外)。
- 自身不做跨源降级、不读缓存(那是 datahub 的事)。
- 任何异常吞掉返空(空 DataFrame/`{}`/`[]`),绝不抛(让 `_route` 切下一源)。
- `if __name__ == '__main__':` 自测一两个样例,直连可达性自检。

## 11. 迁移范式示例(阶段 1 第一刀,可照搬)

以「删 K线 qfq 的 akshare 二道源」为例,演示**换源不动 datahub API**的标准做法:

**改前**(`datahub.py` qfq 链含 akshare 二道东财):
```python
_srcs_q = [_sina_q, _bao_q, _east_q, _aksh_q, _tick_q]   # _aksh_q = ak.stock_zh_a_hist(其实也是东财)
df = _route("kline_qfq", _srcs_q, ...)
```
**改后**(去掉 `_aksh_q`,东财已有 `_east_q` 直连;akshare 若留则降到末位):
```python
_srcs_q = [_sina_q, _bao_q, _east_q, _tick_q]            # 删冗余的 akshare 东财二道
# 若保留 akshare 兜底:_srcs_q = [_sina_q, _bao_q, _east_q, _tick_q, _aksh_q]  # 末位
df = _route("kline_qfq", _srcs_q, ...)
```
**验证**(范式,每刀都跑):
```python
# 1) 该域回归:改前后输出逐字段一致
import datahub
before = datahub.kline('600519','1y',adjust='qfq', use_cache=False)   # 改前先存基线
after  = datahub.kline('600519','1y',adjust='qfq', use_cache=False)   # 改后对照 len/列/末行
# 2) 健康度路由仍正常:多只票、断源场景 _route 正确降级
```
**要点**:datahub 域函数签名、返回格式、调用方**全程零改动**——只动 `_route` 的源列表。这就是"换源不动门面"的范式,阶段 1-2 每一刀都照此办。
