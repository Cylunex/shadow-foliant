> A股多智能体分析/选股/盯盘/回测平台。本文件只记**当前状态与关键约定**(历史改动日志已移除)。
> 能力全景看 [ARCHITECTURE.md](ARCHITECTURE.md);Agent/MCP 能力看 [AGENT_SKILL.md](AGENT_SKILL.md);安装运行看 [使用文档.md](使用文档.md)。

## 怎么跑
- 本机开发:`python webui/run_dev.py` → http://localhost:8601(改前端 no-store 即时生效;改 .py 需重启)。
- 生产:`uvicorn webui.api_server:app --port 8601`(走 .env,USE_POSTGRES=true → 真 PG)。
- 定时任务:supervisor 常驻 `register_default_jobs(); serve_forever()`(生产服务器目录,venv2)。
- MCP(Agent 接入,58+ 工具):`python mcp_server.py`。

## 关键约定(动代码前必读)
1. **统一数据层 `data/datahub.py` 是取外部数据的唯一门面**。一律 `import datahub` 调用(quotes/kline/indices/north_flow/dragon_tiger/sector/financials/valuation/news/screen/convertible_bonds…),**别直接 import a_stock_data_adapter / data_source_manager / 拼外部域名**(那些是 datahub 底下的内部源)。加新源 = 往对应域 `_route(...)` 加一项。
2. **datahub 三级缓存**:L0 进程内存 → L1 Redis(JSON友好值)→ L2 本地文件 pickle(含 DataFrame)。每域按时效配 TTL;`use_cache=False` 或 env `DATAHUB_CACHE=false` 绕过;`datahub.cache_stats()`/`cache_clear()` 观测清理。K线另有专用磁盘缓存 `db/kline_cache`(盘后 `kline_prefetch` 任务焐热)。
3. **jobs 唯一调度入口 `jobs_hub._wrap`**:REGISTRY 内任务按 `automation_config` 开关闸门控;**任务抛异常 → job_runs 记 error(带 traceback)+ 推告警(默认QQ,2h限频)**。每日市场/分析任务都有 `_skip_if_not_trading` 守卫;weekly/周末任务(weekly_analysis 周日、ai_eval_weekly 周一、mx_weekend_outlook 周日)**不守卫**(周末/假日照跑)。
4. **盈利闭环**:选股(unified_selection / wf_daily_strategy_scan)→ 入推荐池(save_recommendation,记 ref_price)→ 盘中追价(ai_rec_check 回填 last_price,候选也批量刷价)→ 周评估(ai_eval_weekly 真实胜率)→ 回喂门槛(_source_feedback)。买入无止损按 entry×(1-8%) 兜底保证盈亏比可校验。
5. **任务默认全开**:automation_config 内任务 default 现全 True。生产开关以 DB `automation_switches` 为准(无记录才回退 default)。
