## 怎么跑
- 本机开发:`python webui/run_dev.py` → http://localhost:8601(改前端 no-store 即时生效;改 .py 需重启)。
- 生产:`uvicorn webui.api_server:app --port 8601`(走 .env,USE_POSTGRES=true → 真 PG)。
- 定时任务:supervisor 常驻 `register_default_jobs(); serve_forever()`(生产服务器目录,venv2)。
- MCP(Agent 接入,58+ 工具):`python mcp_server.py`。

## 关键约定(动代码前必读)
1. **统一数据层 `data/datahub.py` 是取外部数据的唯一门面**。一律 `import datahub` 调用(quotes/kline/indices/north_flow/dragon_tiger/sector/financials/valuation/news/screen/convertible_bonds…),**别直接 import a_stock_data_adapter / data_source_manager / 拼外部域名**(那些是 datahub 底下的内部源)。加新源 = 往对应域 `_route(...)` 加一项。
2. **datahub 三级缓存**:L0 进程内存 → L1 Redis(JSON友好值)→ L2 本地文件 pickle(含 DataFrame)。每域按时效配 TTL;`use_cache=False` 或 env `DATAHUB_CACHE=false` 绕过;`datahub.cache_stats()`/`cache_clear()` 观测清理。K线另有专用磁盘缓存 `db/kline_cache`(盘后 `kline_prefetch` 任务焐热)。
3. **jobs 唯一调度入口 `jobs_hub._wrap`**:REGISTRY 内任务按 `automation_config` 开关闸门控;**任务抛异常 → job_runs 记 error(带 traceback)+ 推告警(默认QQ,2h限频)**。每日市场/分析任务都有 `_skip_if_not_trading` 守卫;weekly/周末任务(weekly_analysis 周日、ai_eval_weekly 周一、mx_weekend_outlook 周日)**不守卫**(周末/假日照跑)。
4. **盈利闭环**:选股(unified_selection / wf_daily_strategy_scan)→ 入推荐池(save_recommendation,记 ref_price)→ 盘中追价(ai_rec_check 回填 last_price,候选也批量刷价)→ 周评估(ai_eval_weekly 真实胜率)→ 回喂门槛(_source_feedback)。买入无止损按 entry×(1-8%) 兜底保证盈亏比可校验。
5. **任务默认全开**:automation_config 内任务 default 现全 True。生产开关以 DB `automation_switches` 为准(无记录才回退 default)。

## 组合级回测引擎(已完成)
单股回测 `analysis/backtest_engine.py` 各信号独立全仓→**系统性高估收益**;组合回测才是实盘口径。
- **`analysis/portfolio_backtest.py`**:事件驱动 + 单一现金账户 + `max_positions` 并发上限 + 等权/信号强度分配。
  每根 bar **先卖**(止损/止盈/到期)释放现金**再买**;成本拆佣金(万2.5,单边最低5元)+印花税(卖0.1%)+滑点(0.05%/边);
  输出组合 CAGR/最大回撤/夏普/年化波动/胜率/盈亏比/平均仓位/净值曲线,对比沪深300(超额收益)。
  **无前视**:信号在 bar t 判定,t+1 开盘建仓;末日按 close 强制平仓使口径干净。已验证:现金守恒、并发不超上限、入场价=触发次日开盘×(1+滑点)。
  - 入口:`portfolio_backtest(stocks, start, end, strategy_id=..., ...)`;`portfolio_backtest_live(...)` 用基因组 live 集(与实盘选股 evolved=True 同口径)。
  - 复用 `datahub.kline` + `backtest_engine._trigger_dates`(信号/参数化变体完全复用);基准走 akshare 指数日线→datahub 兜底。
- 接入:webui 回测页"🧮 组合级回测"卡片 + **`POST /api/backtest/portfolio`**(universe=holdings/index/custom);MCP 工具 **`portfolio_backtest`**。
- ⚡ **性能**:信号计算是 CPU 密集(逐日跑策略),用**进程池**并行(实测~3x;线程被 GIL 卡住反而更慢)。仅默认 datahub 源 + 任务量(股数×策略数)≥24 才开进程,小任务串行;自定义 fetcher 始终串行。基准结果 1h 进程内缓存。
- ⚠️ **数据深度受限**:行情主源(新浪日K)单次约返回 ~365 根,故实际回测区间≈近 1~1.5 年,即便传 3y。要做真·多年回测须先在数据层补长历史(kline_prefetch 目前只焐 1y)。
- 可选下一步:jobs 周度组合回测落库 + 趋势看板;数据层接长历史源解除上述深度限制。

## LLM 层加固:不阻塞主路径(2026-06-14)
原则:选股/持仓建议/离场的 LLM 都是**规则结果之上的增强层**,绝不能让挂起的 LLM 阻塞主路径。
- **🔴 核心修复:LLM 无超时**。`core/llm_router.call()` 与 `core/deepseek_client.call_api()` 原用 openai SDK **默认超时 ~10min**——挂起的 provider 会卡死调用方(选股 job 的 `agent_run`、持仓 AI 诊断、离场决策)。已加 **per-call timeout**(env `LLM_TIMEOUT` 默认40s;thinking 取 max(.,120s))+ **`max_retries=0`**(SDK默认重试2次会把超时×3;降级由 router 跨 provider 自己做)。实测:挂起源从~10min→有界(~timeout)即降级。working 调用不受影响(持仓AI诊断正常出结果)。
- **持仓 AI 诊断**(`portfolio_insights.diagnose_portfolio` 原死代码):`GET /api/portfolio/diagnose-ai` + 持仓页"🧠AI持仓诊断"按钮 + MCP `portfolio_ai_diagnose`。LLM 给 交易习惯/风险/改进建议 + 风险/纪律评分;**无 key/超时不阻塞**——仍返回 context(规则报表),仅成功才缓存30min,60s 截止护栏。顺带把 `portfolio_valuation` 的逐只报价改 `datahub.quotes` **批量**(N→1)。
- 选股 job(`jobs_hub.wf_daily_strategy_scan`)结构已正确:规则扫描(scan_results)先于 LLM,TOP5 才 `agent_run`,每只 try/except 互不影响;现叠加 per-call 超时,单只 LLM 卡住不再拖垮整 job。
- 注:本机**已配 DeepSeek key**(LLM 实际可用,非降级)。生产同样受这套超时保护。
