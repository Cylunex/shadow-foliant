# shadow-foliant — 项目现状 · 运维要点 · 下一步

> A股多智能体分析/选股/盯盘/回测平台。本文件只记**当前状态与关键约定**(历史改动日志已移除)。
> 能力全景看 [ARCHITECTURE.md](ARCHITECTURE.md);Agent/MCP 能力看 [AGENT_SKILL.md](AGENT_SKILL.md);安装运行看 [使用文档.md](使用文档.md)。

## 怎么跑
- 本机开发:`python webui/run_dev.py` → http://localhost:8601(改前端 no-store 即时生效;改 .py 需重启;已内置带时间戳日志)。
- 生产:`uvicorn webui.api_server:app --port 8601 --log-config webui/log_config.json`(走 .env,USE_POSTGRES=true → 真 PG)。`--log-config` 给 uvicorn 日志加时间戳(默认不带)。
- 定时任务:supervisor 常驻 `register_default_jobs(); serve_forever()`(生产服务器目录,独立 venv;**venv 别从别处复制**,见末节坑)。
- MCP(Agent 接入,65+ 工具):`python mcp_server.py`。

## 关键约定(动代码前必读)
1. **统一数据层 `data/datahub.py` 是取外部数据的唯一门面**。一律 `import datahub` 调用(quotes/kline/indices/north_flow/dragon_tiger/sector/financials/valuation/news/screen/convertible_bonds…),**别直接 import a_stock_data_adapter / data_source_manager / 拼外部域名**(那些是 datahub 底下的内部源)。加新源 = 往对应域 `_route(...)` 加一项。
2. **datahub 三级缓存**:L0 进程内存 → L1 Redis(JSON友好值)→ L2 本地文件 pickle(含 DataFrame)。每域按时效配 TTL;`use_cache=False` 或 env `DATAHUB_CACHE=false` 绕过;`datahub.cache_stats()`/`cache_clear()` 观测清理。K线另有专用磁盘缓存 `db/kline_cache`(盘后 `kline_prefetch` 任务焐热)。
3. **jobs 唯一调度入口 `jobs_hub._wrap`**:REGISTRY 内任务按 `automation_config` 开关闸门控;**任务抛异常 → job_runs 记 error(带 traceback)+ 推告警(默认QQ,2h限频)**。每日市场/分析任务都有 `_skip_if_not_trading` 守卫;weekly/周末任务(weekly_analysis 周日、ai_eval_weekly 周一、mx_weekend_outlook 周日)**不守卫**(周末/假日照跑)。
4. **盈利闭环**:选股(unified_selection / wf_daily_strategy_scan)→ 入推荐池(save_recommendation,记 ref_price)→ 盘中追价(ai_rec_check 回填 last_price,候选也批量刷价)→ 周评估(ai_eval_weekly 真实胜率)→ 回喂门槛(_source_feedback)。买入无止损按 entry×(1-8%) 兜底保证盈亏比可校验。
5. **任务默认全开**:automation_config 内任务 default 现全 True。生产开关以 DB `automation_switches` 为准(无记录才回退 default)。

## 关键坑与教训(动这些代码前看)
- **逐只外部抓取的批量函数**(分级/信号/选股)三大坑:①串行 ②用无缓存 `StockDataFetcher`(冷门股冷拉会卡死) ③隐藏 N² 全持仓遍历(典型:原 `position_guardian._get_total_capital` 在 `evaluate_one` 里每只触发股都遍历全持仓拉实时价→~1400次/超时,已改成本价估算降到 2.9s)。接入这类能力必做:走 `datahub` 缓存源、并发、`_DEADLINE_POOL`+超时护栏、按市值 top-N、部分失败不缓存。
- **`webui/_jsonsafe` 已兜底 numpy 标量**:`np.bool_`/`np.float64`(numpy 比较/计算产物)不是 JSON 原生类型,会让 FastAPI 序列化抛 **500**;返回含 numpy 值的端点靠它递归 `.item()` 降级。
- **多源聚合端点按源分缓存 + `keep` 拒空**:否则某源瞬时失败会被连同好数据一起缓存(fund extras、portfolio signals 都踩过;后者用 `_complete`/`add_ok/reduce_ok` 区分"无信号"与"超时")。
- **`instock_strategy_runner._normalize_df` 丢弃 close<=0/NaN 脏 bar**:数据源断线重传偶发 0/NaN → 下游 `(a-close)/close`、`pct_change` 除零/inf → 虚假信号(根治全 13 策略 + 回测)。
- **LLM 必须有超时**:`core/llm_router.call`/`core/deepseek_client.call_api` 已加 per-call timeout(`LLM_TIMEOUT` 默认40s,thinking 120s)+ `max_retries=0`(SDK 默认重试2次会把超时×3)。LLM 是规则结果**之上的增强层**,挂起的 provider 不得阻塞选股/持仓/离场主路径。
- **venv 不可复制**:跨项目/机器复制 venv 必坏(写死绝对路径 + C 扩展按当时 Python 编译,典型症状 `ModuleNotFoundError: anyio._backends`)。规则:一项目一 venv、`python -m venv` 现建、Python 小版本一致;放项目内外皆可。
- **mootdx 已移出主依赖**:其包元数据锁 `httpx<0.26`,与 mcp 需要的 `httpx>=0.27` 冲突 → 一起写进 `requirements.txt` 会让 pip 新解析器 `ResolutionImpossible` 整批装不上。mootdx 是可选 K 线兜底源(`data_source_manager` try/except 守卫,缺库自动跳过),需要时单独装:`pip install "mootdx>=0.11.0" && pip install -U "httpx>=0.27.1"`。
- **日志时间戳**:uvicorn 默认日志不带时间 → `--log-config webui/log_config.json`(run_dev 已内置);项目大量 `print()` 不归 uvicorn 日志管,要给它们加时间需 supervisor 命令里 `| ts`(moreutils,记得 `python -u` + `stopasgroup/killasgroup`)或上 journald。
