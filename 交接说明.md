## 怎么跑
- 本机开发:`python webui/run_dev.py` → http://localhost:8601(改前端 no-store 即时生效;改 .py 需重启;已内置带时间戳日志)。
- 生产:`uvicorn webui.api_server:app --port 8601 --log-config webui/log_config.json`(走 .env,USE_POSTGRES=true → 真 PG)。`--log-config` 给 uvicorn 日志加时间戳(默认不带)。
- 定时任务:supervisor 常驻 `register_default_jobs(); serve_forever()`(生产服务器目录,独立 venv;**venv 别从别处复制**,见末节坑)。
- MCP(Agent 接入,80+ 工具):`python mcp_server.py`。

## 关键约定(动代码前必读)
1. **统一数据层 `data/datahub.py` 是取外部数据的唯一门面**。一律 `import datahub` 调用(quotes/kline/indices/north_flow/dragon_tiger/sector/financials/valuation/news/screen/convertible_bonds/stock_reports/industry_reports/lockup_expiry…),**别直接 import a_stock_data_adapter / data_source_manager / 拼外部域名**(那些是 datahub 底下的内部源)。加新源 = 往对应域 `_route(...)` 加一项。
2. **datahub 三级缓存**:L0 进程内存 → L1 Redis(JSON友好值)→ L2 本地文件 pickle(含 DataFrame)。每域按时效配 TTL;`use_cache=False` 或 env `DATAHUB_CACHE=false` 绕过;`datahub.cache_stats()`/`cache_clear()` 观测清理。K线另有专用磁盘缓存 `db/kline_cache`(盘后 `kline_prefetch` 焐热;`StockDataFetcher` 也共享此缓存)。
3. **jobs 唯一调度入口 `jobs_hub._wrap`**:REGISTRY 内任务按 `automation_config` 开关闸门控;**任务抛异常 → job_runs 记 error(带 traceback)+ 推告警**。每日市场/分析任务都有 `_skip_if_not_trading` 守卫;周末任务(weekly_analysis 周日、mx_weekend_outlook 周日、portfolio_stress_ai 周日、ai_eval_weekly 周一)**不守卫**(周末/假日照跑)。
4. **盈利闭环**:选股(unified_selection / wf_daily_strategy_scan)→ 推荐池(save_recommendation,记 ref_price)→ 盘中追价(ai_rec_check 回填 last_price/realized;了结按委托价 tp/sl 口径)→ 周评估(ai_eval_weekly 真实胜率,零收益不计胜负分母)→ 回喂门槛(_source_feedback,conservative 收紧)。**并行**:所有分析/选股/AI 能力的结论都写 `decision_signal`,16:10 按 K线方向后验,形成可量化的第二闭环。
5. **任务默认全开**:automation_config 内任务 default 现全 True。生产开关以 DB `automation_switches` 为准(无记录才回退 default)。

## 关键坑与教训(动这些代码前看)
- **`get_indicator_snapshot(key)` 直接返回 indicators 列的反序列化结果,没有 'indicators' 外层包裹**:`save_indicator_snapshot(k, {'picks':[...]})` 存什么读出来就是什么。历史上多处误写 `snap.get('indicators')` → 恒 None → 死分支(`_last_selection` 的盘后扫描并入 / 妙想复核 / mx_advisor 全踩过)。读 `_last_selection` 一律走 `jobs_hub._last_selection_picks()`,别裸读。
- **盘后扫描买入判定别用 `'buy' in text` 子串**:plan_execute 自由文本里"不建议买/avoid/选项回显"会被误判为买入、污染推荐池与胜率。已改中文正向词 + 否定优先门;`decision_signal.normalize_action` 也是否定优先。
- **批量解析 LLM 逐行结论用 `re.search` 不用 `re.match`**:LLM 偶尔给 `- `/`1.`/`·` 前缀,行首锚定会整批静默丢空。
- **逐只外部抓取的批量函数**三大坑:①串行 ②用无缓存源冷拉卡死 ③隐藏 N² 全持仓遍历。接入必做:走 `datahub` 缓存源、并发、超时护栏、按市值 top-N、部分失败不缓存、批量取价(`datahub.quotes` 一次拉)。
- **`webui/_jsonsafe` 已兜底 numpy 标量**:`np.bool_`/`np.float64` 不是 JSON 原生类型,会让 FastAPI 序列化抛 500;返回含 numpy 值的端点靠它递归 `.item()` 降级。
- **多源聚合端点按源分缓存 + 拒空**:否则某源瞬时失败会被连同好数据一起缓存。
- **了结/胜率口径**:推荐到期了结必置 `is_active=0`(否则 list_active 反复捞回);止盈止损 realized 用委托价 tp/sl(非快照价,跳空会失真);评估零收益样本不计胜负分母。
- **LLM 必须有超时**:`llm_router.call`/`deepseek_client.call_api` 已加 per-call timeout(`LLM_TIMEOUT` 默认40s,thinking 120s)+ `max_retries=0`。LLM 是规则之上的增强层,挂起 provider 不得阻塞主路径。
- **venv 不可复制**:跨项目/机器复制必坏。一项目一 venv、`python -m venv` 现建、Python 小版本一致。
- **mootdx 已移出主依赖**(httpx pin 与 mcp 冲突):可选 K线兜底源,需要时单独装 `pip install "mootdx>=0.11.0" && pip install -U "httpx>=0.27.1"`。
- **日志时间戳**:uvicorn 默认不带时间 → `--log-config webui/log_config.json`(run_dev 已内置);大量 `print()` 加时间需 supervisor `| ts` 或 journald。
